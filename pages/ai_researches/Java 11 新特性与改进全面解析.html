一、Java 11 语言特性的革新

局部变量类型推断的增强

Java 11允许在隐式类型lambda表达式的形式参数中使用 var，使lambda参数声明与局部变量声明在语法上保持一致。

// Java 10中合法
var list = new ArrayList<String>();

// Java 11新特性
Consumer<String> consumer = (var i) -> System.out.println(i);

// 更方便地为lambda参数添加注解
Predicate<String> tooLong = (@Nonnull var s) -> s.length() > 42;

单行程序运行的革命性改进

Java 11引入了直接运行单文件源代码程序的能力，支持shebang机制，允许在Linux和macOS系统上将Java程序作为脚本执行。

// 直接运行Java源文件
java HelloWorld.java

// 使用shebang机制（Linux/macOS）
#!/usr/bin/java --source 11
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java 11!");
    }
}

Nest-Based Access Control

将逻辑上相关但编译成不同类文件的类定义为一个nest（巢），nest成员可以访问彼此的私有成员而无需编译器插入桥接方法。

public class Outer {
    private int outerInt;
    
    class Inner {
        public void printOuterInt() {
            // 直接访问私有成员，无需桥接方法
            System.out.println("Outer int = " + outerInt);
        }
    }
}

字符串处理能力的全面提升

Java 11为String类添加了多个实用方法，如 isBlank()、lines()、repeat(int) 和 strip() 系列方法。

String blank = "\t\n";
System.out.println(blank.isBlank());    // true

String poem = "Roses are red\nViolets are blue";
poem.lines().forEach(System.out::println);

String repeated = "Ha".repeat(3); // "HaHaHa"

String str = "   Hello, World!   ";
System.out.println(str.strip());      // "Hello, World!"

二、标准库的重大更新与改进

HTTP客户端的标准化

Java 11将JDK 9中引入的孵化HTTP客户端API标准化，提供了对HTTP/1.1和HTTP/2的高级客户端接口，以及对WebSocket的支持。

HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/data"))
    .timeout(Duration.ofSeconds(10))
    .build();

client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println)
    .join();

Files类的便捷方法

Java 11为Files类添加了 readString() 和 writeString() 方法，极大地简化了文件的读写操作。

// 读取文件内容
String content = Files.readString(Paths.get("data.txt"));

// 写入文件内容
Files.writeString(Paths.get("output.txt"), "Hello, World!");

// 指定字符集
Files.writeString(Paths.get("output.txt"), "Hello, World!", 
    StandardCharsets.UTF_8, StandardOpenOption.APPEND);

Java EE和CORBA模块的移除

Java 11移除了Java EE和CORBA模块，这些模块自JDK 9起就被标记为废弃。Java EE已转移到Eclipse Foundation，成为Jakarta EE。

全新的Null Stream类

Java 11添加了四个新的静态工厂方法，用于创建丢弃所有数据的流，类似于Unix系统中的 /dev/null。

// 返回一个不读取任何字节的输入流
InputStream.nullInputStream()

// 返回一个丢弃所有字符的写入器
Writer.nullWriter()

三、性能优化的重大突破

ZGC：革命性的低延迟垃圾回收器

ZGC是一个可扩展的低延迟垃圾回收器，设计目标是GC停顿时间不超过10毫秒，处理堆大小从几百MB到数TB不等。

平均停顿时间 1.091ms
99% 停顿时间 1.512ms 
// 启用ZGC（Java 11中为实验性特性）
-XX:+UnlockExperimentalVMOptions -XX:+UseZGC

GC停顿时间对比 (ms)

Epsilon：无操作垃圾回收器

Epsilon只处理内存分配，不实现任何实际的内存回收机制。当可用Java堆耗尽时，JVM将关闭并抛出 OutOfMemoryError。

// 启用Epsilon GC
-XX:+UseEpsilonGC

// 应用场景：
// 1. 性能基准测试
// 2. 内存压力测试
// 3. 不需要GC的特殊场景

Flight Recorder的开源化

Java 11将Flight Recorder开源，使其从Oracle JDK的商业特性转变为OpenJDK的一部分。这是一个轻量级的HotSpot JVM工具，用于收集诊断和分析数据。

低开销的数据收集框架
故障排除Java应用程序和JVM
连续运行而对性能影响最小

四、安全性的全面增强

TLS 1.3协议的全面支持

Java 11实现了TLS 1.3协议，这是传输层安全协议的最新版本，带来了多项安全和性能改进，包括减少握手往返次数和增强的密码套件。

握手往返次数从2-3次减少到1次
禁用已知有风险的加密算法
支持TLS_AES_128_GCM_SHA256等密码套件

ChaCha20和Poly1305加密算法

Java 11实现了ChaCha20和ChaCha20-Poly1305加密算法，这些算法是基于RFC 7539标准实现的，提供了更强的加密能力和更好的性能。

// 生成256位密钥
KeyGenerator keyGen = KeyGenerator.getInstance("ChaCha20");
keyGen.init(256);
SecretKey key = keyGen.generateKey();

// 加密
Cipher encryptCipher = Cipher.getInstance("ChaCha20-Poly1305");
encryptCipher.init(Cipher.ENCRYPT_MODE, key, new IvParameterSpec(nonce));
byte[] ciphertext = encryptCipher.doFinal(plaintext.getBytes());

Curve25519和Curve448密钥协议

Java 11实现了基于Curve25519和Curve448的密钥协议，这些椭圆曲线算法是基于RFC 7748标准实现的，提供了常数时间实现和高安全性。

// 初始化密钥协议
KeyAgreement keyAgree = KeyAgreement.getInstance("X25519");

// 生成密钥对
KeyPairGenerator kpg = KeyPairGenerator.getInstance("X25519");
kpg.initialize(255);
KeyPair kp = kpg.generateKeyPair();

七、实践案例与最佳实践

Web应用开发案例

以下是一个使用Java 11新特性构建的简单Web应用示例，展示了HTTP客户端API、异步编程模型和lambda表达式增强等特性。

import java.net.http.*;
import java.net.URI;
import java.time.Duration;
import java.util.concurrent.CompletableFuture;

public class WebApplication {
    private final HttpClient httpClient;
    
    public WebApplication() {
        httpClient = HttpClient.newBuilder()
            .version(HttpClient.Version.HTTP_2)
            .connectTimeout(Duration.ofSeconds(10))
            .build();
    }
    
    public CompletableFuture<String> fetchData(String url) {
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(url))
            .header("Accept", "application/json")
            .build();
        
        return httpClient.sendAsync(request, 
            HttpResponse.BodyHandlers.ofString())
            .thenApply(HttpResponse::body);
    }
    
    public static void main(String[] args) {
        WebApplication app = new WebApplication();
        app.fetchData("https://api.example.com/data")
            .thenAccept(data -> System.out.println("Received data: " + data))
            .exceptionally(throwable -> {
                System.err.println("Error fetching data: " + 
                    throwable.getMessage());
                return null;
            });
    }
}

命令行工具开发案例

以下是一个使用Java 11单文件执行特性的命令行工具示例，展示了Files类新方法、字符串处理增强和Stream API的使用。

#!/usr/bin/java --source 11

import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Paths;
import java.util.stream.Stream;

public class FileProcessor {
    public static void main(String[] args) {
        if (args.length != 1) {
            System.err.println("Usage: ./fileprocessor <filename>");
            System.exit(1);
        }
        
        try (Stream<String> lines = Files.lines(Paths.get(args[0]))) {
            var stats = lines.collect(
                () -> new int[3], // [lines, words, chars]
                (a, line) -> {
                    a[0]++;
                    a[1] += line.split("\\s+").length;
                    a[2] += line.length();
                },
                (a, b) -> {
                    a[0] += b[0];
                    a[1] += b[1];
                    a[2] += b[2];
                }
            );
            
            System.out.printf("Lines: %d%nWords: %d%nChars: %d%n", 
                stats[0], stats[1], stats[2]);
        } catch (IOException e) {
            System.err.println("Error processing file: " + e.getMessage());
        }
    }
}
